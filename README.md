# 评分约定
0分：史，无需多言，只是史
1分：有巨大缺点，恶性bug，或根本玩不下去，体验还不如纯乱塞模组的水槽包
2分：有明显缺点 恶性bug，体验接近于纯乱塞模组的水槽包
3分：乱塞模组的水槽包，模组之间根本不兼容
4分：水槽包，但是模组至少还能互相兼容，但依然有乱塞的嫌疑
5分：体验接近于原生包，有着至少逻辑自洽的mod组合
6分：体验较好的水槽或原生包；魔改较少但没有明显缺点，且魔改能感觉到改善了体验的包。
7分：某方面有明显优点的魔改包；较为创新且没有因为创新衍生出各种问题的包；或全程平滑没有明显缺点玩起来较为舒服的魔改包
8分：有着明显优点且其它方面的处理也较好的魔改包，并且很少有恶性bug，崩溃问题。
9分：全方面都处理得非常优秀的魔改包，几乎没有明显的bug与优化问题。或是虽然有着小缺点但优点极其明显，创新度拉满，开创一类整合包先河的魔改包。
10分：载入史册的经典，极大影响了之后的任何同类型整合包。没有明显的恶性bug与优化问题，全方面都处理得无懈可击。

## 加分项
1.能感觉到技术力很强
2.优化非常好，没有明显bug和掉帧
3.有创新性且没有因创新性引发巨大问题
4.胎教级教程或任务线，全方面教学了整合包的方方面面。
5.整合包有着非常活跃且正面的社区，对玩家的游玩有很大帮助。
## 减分项
1.有内鬼模组，其它模组正常但某个模组导致崩溃/掉帧/体验大幅度下降
2.作者教你玩游戏，某些任务线故意不给最优解让你走更麻烦的办法。
3.掉帧严重，且无法用针对性优化模组大幅改善。
4.魔改内容单纯只有增减合成台配方，感觉到没什么技术力。或是魔改内容很少能体验，对模组之间的融合几乎起不到什么帮助。
5.体验太过单薄，模组少的同时没有主题化某个模组。
6.魔改没有改善体验反而降低体验
7.整合包社区非常负面，一片混乱。或者是作者干脆直接在整合包里嘲讽玩家。
## 玩家
玩家社区测评，增加兴趣，减少疲劳，故而字段越少越好，只填关键信息。
- 整合包名称

国内常用中文名，国外全名，不要缩写，对小白不友好
例如：宝藏猎人3 不要宝猎 宝猎3 
没有通用译名 就用官方的名称
